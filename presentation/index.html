# Développement web moderne
## Les applications web modernes ###### - Les exigences des utilisateurs finaux ont été évolués. Les gens sont fatigués de sites lents et non réactifs. Ils veulent: * Avoir une belle conception * Cesser d'attendre le chargement des pages * Avoir des mises à jour dynamiques de fragments de page --- ### _Besoin d'utiliser les dernières technologies HTML5/CSS3/JavaScript_
## La productivité de développeur ###### - * Ces sites doivent être livrés _rapidement_. * Attendre 1 minute pour « déployer » votre application n’est plus acceptable --- ### _Besoin de bons outils pour ce travail!_
## Logiciel de prête-production ###### - * Les sites web modernes devraient être en mesure de gérer un très grand nombre d'utilisateurs simultanés * Utilisant toutes les applications RESTful massives, qui frapperont fort vos serveurs back-end --- ### _besoin de serveurs robustes, évolutifs, et de haute performance_

## Bower ###### - ### Gestionnaire de paquets JavaScript * Vous pouvez rechercher, installer et mettre à jour vos bibliothèques JavaScript * Il fournit un _grand_ nombre de bibliothèques ``` > bower install angular ```

#Back-End ###### - ### Maven, Spring, Spring MVC REST, Spring Data JPA, Netflix OSS
## Maven / Gradle ###### - ### Les plus populaires Java build tools * Bien connus, des répertoires et des objectifs prédéfinis * Intégration avec les EDIs principaux * Grand écosystème de plug-in, y compris Spring Boot et Yeoman ``` ./mvnw spring-boot:run ``` ``` ./gradlew bootRun ```
## Spring Boot ###### - ### Standard de-facto pour les apps Java * IoC, AOP et abstractions pour faciliter le codage d'application * Rapide et léger, et extrêmement puissant en cas de _besoin_ * Live-reload d'application ```java @Service @Transactional public class UserService { @Inject private UserRepository userRepository; } ```
## Netflix OSS ###### - ### Fournit des outils et des frameworks pour microservices * Eureka - Service _registry_ pour l'équilibrage de charge et de basculement. * Zuul - Proxy pour le routage dynamique, la surveillance et la sécurité * Ribbon - software load balancing pour les services
## Liquibase ###### - ###Les mises à jour de base de données en toute simplicité * Mettre à jour un schéma de base de données * Fonctionne très bien avec Spring et JPA * A besoin d'une bonne compréhension des bases de données et d’ORM
## JPA ###### - ### La solution ORM standard pour Java * Hibernate est utilisé en dessous * Très complet, ensemble impressionnant de fonctionnalités ```java @Entity public class User implements Serializable { @Id private String login; } ```
## Spring Data JPA ###### - ### Complément suplimentaire de JPA * Génère automatiquement des _repositories_ JPA * Supprime beaucoup de code réutilisable ```java public interface PersistenceAuditEventRepository extends JpaRepository <PersistentAuditEvent, String> { List<PersistentAuditEvent> findByPrincipalAndAuditEventDateGreaterThan( String principal, LocalDateTime after); } ```
## MongoDB ###### - ### NoSQL pour l'évolutivité * Alternative à la base de données SQL et le choix JPA * Code similaire: fonctionne comme Spring Data JPA * Plus évolutif, facile à utiliser
## Cassandra ###### - ### NoSQL pour l'évolutivité * 2ème alternative à la base de données SQL et le choix JPA * Haute disponibilité * Des performances extrêmes et une faible latence pour les applications critiques
## Caching ###### - ### Caching est le roi des performances * Ehcache est la solution la plus largement utilisée * HazelCast est une excellente alternative, avec le soutien de clustering! * Hibernate, 2ème niveau de cache ou l’abstraction Spring Caching ```java @Entity @Cache(usage = CacheConcurrencyStrategy.NONSTRICT_READ_WRITE) public class User implements Serializable { } ```
## Spring Security ###### - ### Framework de securité _Blindé_ * L’outil principal, largement utilisé dans toutes les industries * Il sécurise les URL et le code Java (Spring Beans) * Fonctionne bien avec toutes les architectures d'applications Web
## Spring MVC REST ###### - ### Le meilleur REST framework pour Java * Solution REST complète de haute performance * Grand support de Spring , grand support de test ```java @RestController public class AccountResource { @RequestMapping(value = "/app/rest/account", method = RequestMethod.POST) public void saveAccount(@RequestBody UserDTO userDTO) { userService.updateUserInformation(userDTO); } } ```
## Thymeleaf ###### - ### Moteur de template moderne * Utilisé quand le _Single Web Page_ ne suffit pas * Replace JSP et JSTL, qui sont dépassées * Grands modèles pour les concepteurs Web ``` html ‹h1 th:text="#{title}">Hello, world ‹/h1› ```
## Monitoring ###### - ### Prêt pour la production avec Metrics * Surveille la JVM, serveur d'application, beans Spring, cache, et encore plus! * JMX ou graphite rapports ```java @Timed public void saveAccount(@RequestBody UserDTO userDTO) { userService.updateUserInformation(userDTO); } ```
## Cloud deployment ###### - ### Au cloud et au-delà! * Déploiements devrait être facile sur site et dans le cloud * Les fournisseurs de cloud de premier plan comme Cloud Foundry, Heroku et Amazon Web Services offrent tout l'outillage et les options de déploiement spécifiques

## Prerequis ###### - * Installation des outils: Java, Maven / Gradle, Git, Node.js, Bower, Yeoman and Gulp / Grunt * Modification du classpath : JAVA_HOME, MVN_HOME ...etc
## Installation Jhipster ###### - ``` npm install -g generator-jhipster ```
## Generer application ###### - ``` yo jhipster ```
## Front-end live reload ###### - * Reload en direct du front-end (HTML, CSS, JavaScript) grâce à Gulp. * Modifier un fichier et votre navigateur se rafraîchira automatiquement ``` gulp serve ```
## Ajouter une entité ###### - ``` yo jhipster:entity foo ```
## Packaging pour production ###### - ``` mvn -Pprod package java -jar target/*.war --spring.profiles.active=prod ```
## Passer en mode production ###### - ``` yo jhipster:heroku ```
