                    #Back-End 
                    ###### -
                    ### Maven, Spring, Spring MVC REST, Spring Data JPA, Netflix OSS
                </section>
                <section data-markdown>
                    ## Maven / Gradle
                    ###### -
                    ### Les plus populaires Java build tools

                    * Bien connus, des répertoires et des objectifs prédéfinis
                    * Intégration avec les EDIs principaux
                    * Grand écosystème de plug-in, y compris Spring Boot et Yeoman

                    ```

./mvnw spring-boot:run ./gradlew bootRun ``` ## Spring Boot ###### - ### Standard de-facto pour les apps Java 

                    * IoC, AOP et abstractions pour faciliter le codage d'application
                    * Rapide et léger, et extrêmement puissant en cas de _besoin_
                    * Live-reload d'application


                    ```java

@Service @Transactional public class UserService {

@Inject
private UserRepository userRepository;

} ``` ## Netflix OSS ###### - ### Fournit des outils et des frameworks pour microservices 

                    * Eureka - Service _registry_ pour l'équilibrage de charge et de basculement.
                    * Zuul - Proxy pour le routage dynamique, la surveillance et la sécurité
                    * Ribbon - software load balancing pour les services
                </section>
                <section data-markdown>
                    ## Liquibase
                    ###### -
                    ###Les mises à jour de base de données en toute simplicité

                    * Mettre à jour un schéma de base de données
                    * Fonctionne très bien avec Spring et JPA
                    * A besoin d'une bonne compréhension des bases de données et d’ORM
                </section>
                <section data-markdown>
                    ## JPA
                    ###### -
                    ### La solution ORM standard pour Java

                    * Hibernate est utilisé en dessous
                    * Très complet, ensemble impressionnant de fonctionnalités

                    ```java

@Entity public class User implements Serializable {

@Id
private String login;

} ``` ## Spring Data JPA ###### - ### Complément suplimentaire de JPA 

                    * Génère automatiquement des _repositories_ JPA
                    * Supprime beaucoup de code réutilisable

                    ```java

public interface PersistenceAuditEventRepository extends JpaRepository <PersistentAuditEvent, String> {

List&lt;PersistentAuditEvent&gt;
    findByPrincipalAndAuditEventDateGreaterThan(
                String principal, LocalDateTime after);

} ``` ## MongoDB ###### - ### NoSQL pour l'évolutivité 

                    * Alternative à la base de données SQL et le choix JPA
                    * Code similaire: fonctionne comme Spring Data JPA
                    * Plus évolutif, facile à utiliser
                </section>
                <section data-markdown>
                    ## Cassandra
                    ###### -
                    ### NoSQL pour l'évolutivité

                    * 2ème alternative à la base de données SQL et le choix JPA
                    * Haute disponibilité
                    * Des performances extrêmes et une faible latence pour les applications critiques
                </section>
                <section data-markdown>
                    ## Caching
                    ###### -
                    ### Caching est le roi des performances

                    * Ehcache est la solution la plus largement utilisée
                    * HazelCast est une excellente alternative, avec le soutien de clustering!
                    * Hibernate, 2ème niveau de cache ou l’abstraction Spring Caching

                    ```java

@Entity @Cache(usage = CacheConcurrencyStrategy.NONSTRICT_READ_WRITE) public class User implements Serializable {

} ``` ## Spring Security ###### - ### Framework de securité Blindé 

                    * L’outil principal, largement utilisé dans toutes les industries
                    * Il sécurise les URL et le code Java (Spring Beans)
                    * Fonctionne bien avec toutes les architectures d'applications Web

                </section>
                <section data-markdown>
                    ## Spring MVC REST
                    ###### -
                    ### Le meilleur REST framework pour Java

                    * Solution REST complète de haute performance
                    * Grand support de Spring , grand support de test

                    ```java

@RestController public class AccountResource { @RequestMapping(value = "/app/rest/account", method = RequestMethod.POST) public void saveAccount(@RequestBody UserDTO userDTO) { userService.updateUserInformation(userDTO); } } ``` ## Thymeleaf ###### - ### Moteur de template moderne 

                    * Utilisé quand le _Single Web Page_ ne suffit pas
                    * Replace JSP et JSTL, qui sont dépassées
                    * Grands modèles pour les concepteurs Web

                    ``` html
                        &lsaquo;h1 th:text=&quot;#{title}&quot;>Hello, world &lsaquo;/h1&rsaquo;
                    ```
                </section>
                <section data-markdown>
                    ## Monitoring
                    ###### -
                    ### Prêt pour la production avec Metrics

                    * Surveille la JVM, serveur d'application, beans Spring, cache, et encore plus!
                    * JMX ou graphite rapports

                    ```java

@Timed public void saveAccount(@RequestBody UserDTO userDTO) { userService.updateUserInformation(userDTO); } ``` ## Cloud deployment ###### - ### Au cloud et au-delà! 

                    * Déploiements devrait être facile sur site et dans le cloud
                    * Les fournisseurs de cloud de premier plan comme Cloud Foundry, Heroku et Amazon Web Services offrent tout l'outillage et les options de déploiement spécifiques
                </section>
            </section>

gulp serve </section> <section data-markdown> ## Ajouter une entité ###### - yo jhipster:entity foo ```
